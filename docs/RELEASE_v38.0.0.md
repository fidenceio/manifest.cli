# Release v38.0.0

**Release Date:** 2026-04-04 15:25:17 UTC
**Release Type:** major

## What's New

### Fleet Documentation Orchestration

This major release introduces configurable documentation placement for Fleet-managed polyrepos. Users can now control where and how docs folders are created across a fleet via the `docs:` section in `manifest.fleet.yaml`.

**Key capabilities:**
- **Strategy-based placement** — choose `"fleet-root"`, `"per-service"`, or `"both"` to control where docs are generated
- **Fleet-root docs** — aggregated release notes, changelogs, and index across all services at the fleet root
- **Per-service docs** — each service gets its own docs folder with full single-repo documentation
- **Detail level control** — fleet-root docs can be `"summary"` (aggregated changes) or `"index"` (lightweight version list with links)
- **Granular generation flags** — control which document types are created (release notes, changelog, index, README version)
- **CLI overrides** — `--fleet-only`, `--services-only`, and `--strategy` flags for one-off runs

### New Command: `manifest fleet docs`

```bash
manifest fleet docs                          # Generate per configured strategy
manifest fleet docs generate                 # Same as above
manifest fleet docs status                   # Show docs configuration
manifest fleet docs generate --fleet-only    # Only fleet-root docs
manifest fleet docs generate --strategy both # Override strategy
```

### Ship Workflow Integration

`manifest fleet ship` now includes a documentation generation step between prep and PR creation. Docs generation is non-blocking — warnings are logged but do not halt the workflow.
