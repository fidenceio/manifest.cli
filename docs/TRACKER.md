# Manifest CLI — Tracker

Open work for the Manifest CLI repo, as one flat list.

- Every item names a concrete deliverable and an anchor; the `§X.Y` label is a stable ID (referenced from commits, memory, and the Cloud/workspace trackers — do not renumber).
- Drift policy: when an item ships, **delete it** from this file. Provenance lives in the merge commit and release history.
- Status tag (2026-05-22 enterprise-readiness triage):
  - **T1** — release-blocking; absence allows lost user state, lost git history, or unrecoverable partial fleet state.
  - **T2** — contract integrity; absence allows the safe-by-default contract to drift or bypass.
  - **T3** — coverage, audit, or docs required before declaring the v1 enterprise release done.
  - **DEFER** — real value, but absence causes no safety harm at fleet-of-dozens scale.
  - **CUT** — considered and rejected; kept here so it doesn't reappear by accident. File a fresh item rather than reviving a cut entry verbatim if the reasoning changes.

- **§7.3 Fleet `--depth` unification — one default, one meaning.** **Status:** T3 (feature). Defaults disagree across commands (init=2 "scan guardrail", update/discover=5, plan=`auto`→cap 10, find_fleet_root=10) and the flag's *meaning* is inconsistent. Decided direction: unify all discovery commands (init/update/quickstart/detect) on `--depth N|auto` defaulting to **auto** with ONE safety-cap constant as source of truth; give `--depth` a single meaning; centralize auto-resolution in one helper; make `fleet.mode: auto` use the same path. **OPEN DESIGN FORK (gating — settle before coding):** `auto` resolves literally to the cap (10) today, so naive unification makes depth-10 the default *everywhere* — 5× deeper/slower first scan than init's current 2. Decide: make `auto` genuinely adaptive (deepen-until-repos-found, capped; ~medium-high effort, recursion refactor) vs. keep `auto`=cap (low effort, but slower default scan). Investigation refuted the "silent override" claim — each command explicitly passes its own default. **Also reconcile** the workspace [`../../manifest.fleet.config.yaml`](../../manifest.fleet.config.yaml) `validation:` block (`require_clean_status`/`enforce_dependencies`/`strict`, still present — the §2.8-reconciled claim was stale). **Prereq for §7.4.** **Anchor:** [`modules/fleet/manifest-fleet-detect.sh`](../modules/fleet/manifest-fleet-detect.sh) (depth constants + recursion), [`modules/fleet/manifest-fleet-config.sh`](../modules/fleet/manifest-fleet-config.sh) (`find_fleet_root`, max-search cap), [`modules/fleet/manifest-fleet-plan.sh`](../modules/fleet/manifest-fleet-plan.sh) (`auto`→cap), [`modules/core/manifest-init.sh`](../modules/core/manifest-init.sh).
