# Changelog v45.6.0

**Release Date:** 2026-04-29 22:14:59 UTC
**Release Type:** Minor

## Forgiveness Contract for Config Values

This release lands a uniform "be forgiving at the boundary" pass for every
config value the CLI reads. Whitespace and case errors that used to silently
break behavior are now normalized at the load/dispatch points.

### New Helpers

Added to [`modules/core/manifest-shared-utils.sh`](../modules/core/manifest-shared-utils.sh):

- `_trim_ws <value>` — strips leading and trailing whitespace.
- `normalize_enum_value <value>` — trim + lowercase. The canonical
  pre-comparison transform for any enumerated config value.
- `is_truthy <value>` — accepts `1|true|yes|on` (case-insensitive,
  whitespace-tolerant). Returns 0 (success) on truthy, 1 on anything else.
- `is_falsy <value>` — accepts `0|false|no|off|''` (case-insensitive,
  whitespace-tolerant). Note: not the strict inverse of `is_truthy` — garbage
  values are neither truthy nor falsy.

### Trim at Load

[`load_yaml_to_env`](../modules/core/manifest-yaml.sh) now trims leading and
trailing whitespace from every value as it is loaded. Internal whitespace is
preserved (e.g., `commit_template` containing spaces is unaffected). A YAML
value of `" release_head "` — easy to slip in on copy/paste — used to
silently fail every downstream string comparison; it now works.

### Forgiveness in Enum Dispatch

[`resolve_tag_target_sha`](../modules/git/manifest-git.sh) normalizes the
incoming `MANIFEST_CLI_RELEASE_TAG_TARGET` (trim + lowercase) before the
case-match. `Version_Commit`, `"  release_head  "`, and `Final_Release_Commit`
all resolve correctly. The deprecated alias `final_release_commit` continues
to work and emits a deprecation warning pointing at `release_head`.

### Boolean Dispatch Migrated to `is_truthy`

Five sites swapped from strict `[[ "$x" == "1" ]]` checks to `is_truthy`:

- [`manifest-config.sh`](../modules/core/manifest-config.sh) — 3 sites for
  `MANIFEST_CLI_AUTO_CONFIRM`. The warning text now interpolates the actual
  user value instead of the literal string `"1"`.
- [`manifest-security.sh`](../modules/system/manifest-security.sh) —
  `MANIFEST_CLI_SKIP_SECURITY_REPORT`.
- [`manifest-orchestrator.sh`](../modules/workflow/manifest-orchestrator.sh) —
  `MANIFEST_CLI_INTERACTIVE_MODE`. Replaces a 3-way OR (`true|TRUE|1`) with
  a single helper call.
- `log_deprecated` now suppresses on any truthy
  `MANIFEST_CLI_QUIET_DEPRECATIONS` value.

### Test Coverage

24 new bats tests across:

- [`tests/truthy.bats`](../tests/truthy.bats) — 16 tests covering the
  forgiveness grammar, case-insensitivity, whitespace tolerance, and the
  asymmetry between `is_truthy` and `is_falsy` on garbage values.
- [`tests/yaml.bats`](../tests/yaml.bats) — 2 tests for trim at load
  (including a regression test that internal whitespace is preserved).
- [`tests/tag_target.bats`](../tests/tag_target.bats) — 4 tests for the
  enum dispatch's case- and whitespace-tolerance, plus a stderr-asserted
  warning for unknown values.
- [`tests/deprecation.bats`](../tests/deprecation.bats) — 2 tests proving
  `MANIFEST_CLI_QUIET_DEPRECATIONS` accepts the full forgiveness grammar.

Suite total: 183 passing (was 159).

## Why

This is M5 + M6 from [`docs/TAG_TARGET_REVIEW.md`](TAG_TARGET_REVIEW.md),
expanded into a uniform consistency pass at the user's request: apply the
fix everywhere it is principled, not just where a user reported the bug.
The boundary for "principled" is enumerated config values and booleans;
free-form values (branch names, URLs, paths, regex, commit templates) are
preserved verbatim.
