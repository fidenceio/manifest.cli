## What's New in v39.1.1

This patch fixes the documentation generation pipeline to **preserve user-crafted content**
in README.md, CHANGELOG.md, and docs/INDEX.md during `prep` and `ship` workflows.

### Documentation Generation Fixes

Previously, every `manifest prep` or `manifest ship` would overwrite three files with templates:

| File | Old Behavior | New Behavior |
| ---- | ------------ | ------------ |
| README.md | Prepended a version metadata block | Updates inline version strings only |
| CHANGELOG.md | Replaced entirely with version-specific changelog | Updates version references in place |
| docs/INDEX.md | Replaced entirely with a template | Updates version and release links in place |

The fix ensures that handcrafted documentation structure, formatting, and content survive
the automated release process. Only version numbers and release note filenames are updated.

### How It Works

- **README.md**: If no `## Version Information` section exists, only backtick-quoted version
  strings are updated via sed. The section-replacement logic still works for READMEs that use it.
- **CHANGELOG.md**: If the root changelog contains Keep-a-Changelog-style `[version]` entries,
  it's treated as user-crafted and only version strings are updated. Template-style changelogs
  are still replaced as before.
- **docs/INDEX.md**: If the file exists, version numbers, release note links, and date are
  updated in place. New INDEX.md files are still created from a template.
