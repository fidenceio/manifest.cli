# Release v45.6.0

**Release Date:** 2026-04-29 22:14:59 UTC
**Release Type:** Minor

## What's New

A uniform "be forgiving at the boundary" pass for every config value the CLI
reads. Whitespace and case errors that used to silently break behavior are
now normalized at the load/dispatch points.

### Highlights

- **Forgiving truthy/falsy.** `MANIFEST_CLI_AUTO_CONFIRM=true`,
  `MANIFEST_CLI_QUIET_DEPRECATIONS=yes`, etc., now do what users expect.
  The strict `=="1"` checks are gone; `1|true|yes|on` (case-insensitive,
  whitespace-tolerant) all work.
- **Trim at YAML load.** A trailing space in your config used to silently
  break every downstream string comparison. `load_yaml_to_env` now trims
  every value as it loads. Internal whitespace (e.g., in `commit_template`)
  is preserved.
- **Case- and whitespace-tolerant `release.tag_target`.** `Version_Commit`,
  `" release_head "`, and `Final_Release_Commit` all resolve correctly.
- **Deprecation warning for `final_release_commit`.** The old name from
  v44.10.1 still works but emits a one-time warning pointing at `release_head`.
- **24 new bats tests** lock the contract. Suite is now 183 passing.

### New Helpers

Available in [`modules/core/manifest-shared-utils.sh`](../modules/core/manifest-shared-utils.sh)
for use anywhere in the codebase:

- `is_truthy <value>` — bash success-convention predicate.
- `is_falsy <value>` — accepts `0|false|no|off|''`.
- `normalize_enum_value <value>` — trim + lowercase, the canonical
  pre-comparison transform.
- `_trim_ws <value>` — leading/trailing whitespace strip.

## Why

This is M5 + M6 from [`docs/TAG_TARGET_REVIEW.md`](TAG_TARGET_REVIEW.md),
expanded into a uniform consistency pass at the user's request: apply the
fix everywhere it is principled, not just where a user reported the bug.
The boundary for "principled" is enumerated config values and booleans;
free-form values (branch names, URLs, paths, regex, commit templates) are
preserved verbatim.
